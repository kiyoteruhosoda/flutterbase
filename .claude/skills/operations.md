# Skill: Operations

1. Provide scripts for build, packaging, startup, reset, and migration where useful.
2. Keep local development runnable with one documented command when possible.
3. Use separate services/containers for UI, API, database, workers, or batch jobs as needed.
4. Make reset explicit and parameterized so data deletion is intentional.
5. Run migrations automatically only when the project policy allows it, and log the result.
6. Document commands in `docs/OPERATIONS.md`.
