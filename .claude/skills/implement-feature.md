# Skill: Implement Feature

1. Define the user goal and smallest useful slice.
2. Update `docs/Progress.md` if the work spans multiple steps.
3. Model domain concepts first: entities, value objects, services, and interfaces.
4. Add an Application use case that orchestrates the business operation.
5. Add Infrastructure adapters behind Domain interfaces.
6. Add Presentation entry points that call the use case.
7. Add or update configuration, logs, docs, and tests.
8. Run project checks and move completed progress notes to changelog/history if important.
